# Reliability Maintainer Sprint 46: Shared QueryState Failure Feedback

## Status

Closed and commit-ready.

## Problem

The shared `QueryState` wrapper rendered `error.message` directly into
`ErrorState`. Even though many domains now normalize read failures before they
reach UI, this shared fallback still made future or legacy consumers one
prop-passing mistake away from exposing database, provider, stack, or transport
wording to operators.

## Workflow Spine Protected

Query hook/read failure -> shared `QueryState` wrapper -> formatter-owned
operator copy -> retry-capable `ErrorState`.

## Touched Domains

- Shared platform UI.
- Shared read-path feedback.

## Business Value Protected

RENOZ operators should see stable recovery guidance when a list/detail read
fails, not raw infrastructure wording. This protects cross-domain workflows
where a shared query wrapper may be used for orders, inventory, warranty,
support, finance, users, or future battery OEM surfaces.

## Scope Constraints

- No route, hook, server function, schema, database, tenant, inventory, finance,
  serialized lineage, or query/cache behavior changed.
- `ErrorState` remains a presentational component that renders an explicit
  message string from its caller.
- `QueryState` still preserves normalized `ReadQueryError` messages generated by
  domain hooks/read helpers.
- This slice does not migrate individual domain read states to `QueryState`.

## Changes

- Added `formatQueryStateError` for shared query-state feedback.
- Preserved normalized `ReadQueryError` messages.
- Mapped raw unauthorized, forbidden, not-found, and rate-limited failures to
  stable operator copy.
- Failed closed for raw validation, system, unknown, database, provider, stack,
  and transport failures.
- Replaced direct `message={error.message}` in `QueryState`.
- Updated the `ErrorState` usage example so shared docs no longer model raw
  exception rendering.
- Added a focused unit/source contract for the shared feedback boundary.

## Standards Checked

- Domain ownership: shared UI owns the generic fallback; domain-specific read
  helpers still own richer domain copy before errors reach this wrapper.
- Route -> container/page -> hook -> server function -> schema/database flow:
  unchanged.
- Tenant isolation: unchanged.
- Transactional inventory and finance integrity: unchanged.
- Serialized lineage continuity: unchanged.
- Honest UI states: shared read failures now show safe retry/recovery copy while
  preserving normalized domain messages.
- Query/cache contracts: unchanged.
- Reviewable diff: two shared source files, one focused feedback helper, one
  focused test, and docs only.

## Smells Removed

- Removed shared `QueryState` direct rendering of `error.message`.
- Removed a raw `error.message` example from the shared `ErrorState` docs.

## Smells Deferred

- Finance credit-note read-state wrapping still passes a raw `Error` object to a
  presenter and should be handled as a finance-owned slice.
- Admin user import still has ad hoc CSV parsing/result feedback and should be
  handled as a users/admin workflow slice.
- Direct `ErrorState` consumers outside `QueryState` still need local review
  when they pass raw error strings.
- This slice does not add a lint rule banning raw error-message rendering.
