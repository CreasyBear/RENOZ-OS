## 9. Performance Considerations

### 9.1 Virtual Scrolling

For tables with 1000+ rows, implement virtual scrolling:
- Only render visible rows + buffer
- Maintain scroll position
- Use `@tanstack/react-virtual`

### 9.2 Pagination vs Infinite Scroll

**Pagination (default):**
- Better for data analysis
- Easier navigation to specific pages
- Lower memory usage

**Infinite Scroll (mobile):**
- Better mobile UX
- Continuous browsing
- Consider for activity feeds

### 9.3 Debouncing & Throttling

- Search input: 300ms debounce
- Filter changes: 200ms debounce
- Resize: throttle to animation frames

### 10.3 Filter Functions

```typescript
// Global filter
const globalFilterFn = (row, columnId, filterValue) => {
  const searchableFields = ['name', 'email', 'phone']
  return searchableFields.some(field =>
    row.getValue(field)
      ?.toString()
      .toLowerCase()
      .includes(filterValue.toLowerCase())
  )
}

// Column-specific filters
const statusFilterFn = (row, columnId, filterValue) => {
  if (!filterValue.length) return true
  return filterValue.includes(row.getValue(columnId))
}

const rangeFilterFn = (row, columnId, filterValue) => {
  const value = row.getValue(columnId)
  const [min, max] = filterValue
  return value >= min && value <= max
}
```
