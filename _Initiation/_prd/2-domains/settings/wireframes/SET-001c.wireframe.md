# Wireframe: DOM-SET-001c - Integrate System Defaults into Forms

## Story Reference

- **Story ID**: DOM-SET-001c
- **Name**: Integrate System Defaults into Forms
- **PRD**: memory-bank/prd/domains/settings.prd.json
- **Type**: Form Field Integration Pattern
- **Component Type**: Form fields with default value indicators

## Overview

Use system defaults as initial values in quote and order forms with "Using default" helper text and override capability. Forms show clear visual indication when default values are applied vs overridden.

## UI Patterns (Reference Implementation)

### Input with Default Indicator
- **Pattern**: RE-UI Input
- **Reference**: `_reference/.reui-reference/registry/default/ui/input.tsx`
- **Features**:
  - Standard text/number input with default value indicator below
  - Helper text shows "Using organization default" when value is from system defaults
  - Error state with validation messages when custom override is invalid

### Button (Reset to Default)
- **Pattern**: RE-UI Button
- **Reference**: `_reference/.reui-reference/registry/default/ui/button.tsx`
- **Features**:
  - Secondary variant button for reset action
  - Only appears when field value differs from default
  - Click resets field value back to organization default

### Select with Default State
- **Pattern**: RE-UI Select
- **Reference**: `_reference/.reui-reference/registry/default/ui/select.tsx`
- **Features**:
  - Dropdown for currency and status selection
  - Selected value shows default vs custom state indicator
  - Keyboard navigation with default value highlighting

### Alert Banner
- **Pattern**: RE-UI Alert
- **Reference**: `_reference/.reui-reference/registry/default/ui/alert.tsx`
- **Features**:
  - Warning banner when fallback defaults are used (org defaults unavailable)
  - Non-blocking info banner with retry action
  - Amber color scheme for fallback mode indication

## Accessibility Notes

### Focus Order

1. **Default Field Focus**
   - Tab through form fields normally
   - Default indicator is informational, not focusable
   - Reset button is focusable when override is active

2. **Override Interaction**
   - When user modifies field, indicator changes
   - Reset to default link appears and is tabbable
   - Arrow keys adjust number values

3. **Screen Reader Flow**
   - Field announces current value
   - Then announces default status
   - Reset button is clearly labeled

### ARIA Requirements

```html
<!-- Field with Default Applied -->
<div class="form-field">
  <label for="validity-days">
    Valid For (days) *
  </label>
  <input

    value="30"
    aria-describedby="validity-default validity-help"
```

### Screen Reader Announcements

- Default applied: "Valid For, 30 days, using organization default"
- Override detected: "Custom value entered. Organization default is 30 days."
- Reset: "Reset to organization default. Value now 30 days."
- Fallback used: "Using fallback value. Organization settings unavailable."

### Keyboard Navigation

```
Tab           - Move between fields
Arrow Up/Down - Adjust number values
Enter         - Reset to default (when reset button focused)
Escape        - Cancel field edit, revert to previous value
```

## Animation Choreography

### Default Indicator Transitions

```
DEFAULT TO OVERRIDE:
- Duration: 200ms
- Easing: ease-out
- Sequence:
  1. "Using org default" fades out (100ms)
  2. "Custom value" fades in (100ms)
  3. Reset link slides in from left (200ms)

OVERRIDE TO DEFAULT (Reset):
- Duration: 250ms
- Easing: ease-out
- Sequence:
  1. Field value counter animates to default (150ms)
  2. Reset link slides out (100ms)
  3. Checkmark appears briefly (200ms)
  4. "Using org default" fades in (100ms)
```

### Default Loading Animation

```
LOADING DEFAULTS:
- Duration: continuous until loaded
- Easing: linear
- Animation: Skeleton shimmer on field
- Text: "Loading default..." with spinner

DEFAULT APPLIED:
- Duration: 300ms
- Skeleton dissolves (150ms)
- Value fades in (150ms)
- Default indicator appears (100ms)
```

### Fallback Warning

```
FALLBACK BANNER ENTRY:
- Duration: 250ms
- Easing: ease-out
- Transform: translateY(-100%) -> translateY(0)
- Background: amber pulse once (200ms)
```

### Reset Button

```
RESET BUTTON HOVER:
- Duration: 150ms
- Background: subtle highlight
- Text: underline

RESET ACTION:
- Duration: 300ms
- Value morphs: custom -> default (150ms)
- Checkmark flash: scale(0) -> scale(1.2) -> scale(1) (200ms)
- Green tint: 100ms fade in/out
```

## Component Props Interface

```typescript
interface DefaultValueFieldProps<T> {
  // Field configuration
  label: string;
  name: string;
  value: T;
  defaultValue?: T;
  onChange: (value: T) => void;

  // Default source info
  defaultSource: 'organization' | 'fallback' | 'none';
  isLoadingDefault?: boolean;

  // Override state
  isOverridden: boolean;
  onReset: () => void;

  // Validation
  required?: boolean;
  error?: string;
  helpText?: string;

  // Display options
  formatValue?: (value: T) => string;
  formatDefault?: (value: T) => string;
}

interface UseSystemDefaultsOptions {
  // Which defaults to load
  fields: Array<keyof SystemDefaults>;

  // Fallback values if org defaults unavailable
  fallbacks: Partial<SystemDefaults>;

  // Callback when defaults are loaded
  onLoaded?: (defaults: Partial<SystemDefaults>) => void;

  // Callback when load fails
  onError?: (error: Error) => void;
}

interface UseSystemDefaultsReturn {
  // Current default values
  defaults: Partial<SystemDefaults>;

  // Loading state
  isLoading: boolean;

  // Error state
  error: Error | null;

  // Source of values
  source: 'organization' | 'fallback';

  // Refetch function
  refetch: () => Promise<void>;
}

// Hook usage example
function QuoteForm() {
  const { defaults, isLoading, source } = useSystemDefaults({
    fields: ['quoteValidity', 'taxRate', 'currency'],
    fallbacks: {
      quoteValidity: 30,
      taxRate: 10,
      currency: 'AUD',
    },
  });

  const [validity, setValidity] = useState<number | undefined>();

  const isOverridden = validity !== undefined;
  const displayValue = validity ?? defaults.quoteValidity ?? 30;

  return (
    <DefaultValueField
      label="Valid For (days)"
      name="validityDays"
      value={displayValue}
      defaultValue={defaults.quoteValidity}
      defaultSource={source}
      isLoadingDefault={isLoading}
      isOverridden={isOverridden}
      onChange={setValidity}
      onReset={() => setValidity(undefined)}
      required
    />
  );
}

// Default indicator component
interface DefaultIndicatorProps {
  source: 'organization' | 'fallback' | 'none';
  isOverridden: boolean;
  defaultValue?: string | number;
  onReset?: () => void;
  isLoading?: boolean;
}

// Indicator states display
type IndicatorState =
  | 'loading'           // "Loading default..."
  | 'using-default'     // "Using organization default"
  | 'using-fallback'    // "Using fallback value"
  | 'overridden'        // "Custom (default: X)"
  | 'none';             // No indicator (no default configured)
```

## Integration Pattern

### Form Integration Example

```typescript
// In quote form component
import { useSystemDefaults } from '@/hooks/use-system-defaults';
import { DefaultValueField } from '@/components/shared/default-value-field';

function QuoteCreateForm() {
  const { defaults, isLoading, source, error } = useSystemDefaults({
    fields: ['quoteValidity', 'taxRate', 'currency', 'paymentTerms'],
    fallbacks: FALLBACK_DEFAULTS,
  });

  const form = useForm({
    defaultValues: {
      validityDays: undefined, // Will use system default
      taxRate: undefined,      // Will use system default
      // ... other fields
    },
  });

  // Show fallback warning if needed
  const showFallbackWarning = source === 'fallback' && !error;

  return (
    <Form {...form}>
      {showFallbackWarning && (
        <Alert variant="warning">
          Using fallback defaults. Organization settings unavailable.
        </Alert>
      )}

      <FormSection title="Quote Details">
        <DefaultValueField
          label="Valid For (days)"
          value={form.watch('validityDays') ?? defaults.quoteValidity}
          defaultValue={defaults.quoteValidity}
          defaultSource={source}
          isLoadingDefault={isLoading}
          isOverridden={form.watch('validityDays') !== undefined}
          onChange={(v) => form.setValue('validityDays', v)}
          onReset={() => form.setValue('validityDays', undefined)}
          required
        />
      </FormSection>

      <FormSection title="Financial">
        <DefaultValueField
          label="Tax Rate (%)"
          value={form.watch('taxRate') ?? defaults.taxRate}
          defaultValue={defaults.taxRate}
          defaultSource={source}
          isLoadingDefault={isLoading}
          isOverridden={form.watch('taxRate') !== undefined}
          onChange={(v) => form.setValue('taxRate', v)}
          onReset={() => form.setValue('taxRate', undefined)}
        />
      </FormSection>
    </Form>
  );
}
```
